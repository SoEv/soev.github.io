Техн. заметки

Технические заметки

  • офиц. документация: toolchain.gitbook.com/structure.html
  • shell: create PDF
    # Generate a PDF file
$ gitbook pdf ./ ./mybook.pdf

  • shell: build the static website

    $ gitbook build

  • переходим в папку _book

      # Создаём совершенно новый репозиторий с чистой историей
  git init

  # указываем наш репозиторий
  git remote add origin https://SoEv@bitbucket.org/SoEv/gitbk.git

  # Посмотрим, какие у нас ветки, и нет ли там ветки с именем `book`
  git branch

  # создаём в локальном репо новую ветку `book`
  git branch book

  # переключаемся в локальном репо на ветку `book`
  git checkout book

  # в локальном репо удаляем `master`
  git branch -D master

  # если на GitHub по умолчанию `master`, то на попытку удалить: 
  # `git push origin :master` на GitHub, получим отказ.

  # пока не найден другой способ. На этом шаге ужно идти на GitHub и сделать
  # ветку по умолчанию не `master`, а `book`
  # https://github.com/SoEv/soev.github.io/settings/branches  

  # после этого команда удалить ветку на GitHub проходит чисто:
  git push origin --delete master

  # запихиваем в репозиторий на GitHub построенное дерево сайта
  git push -u origin book

  # переименовываем в личном репо book в master
  git branch -m book master

  # Состояние:
  # на GitHub 2 branches, по умолчанию назначена ветка `book`
  # ветка `master` присутствует, сайт работает
  #
  # локально: 1 branch, `master`

  # удалить ветку `book` на GitHub не получится теперь, по той же причине, что
  # и `master`: она стоит по умолчанию и обслуживает web-интерфейс GitHub
  # git push origin --delete book

  # Можно посмотреть, какие ветки есть на GitHub и какие у нас локально:
  git branch -a

  # создаём на GitHub новую ветку `__tempo` и делаем её по умолчанию.
  # после этого команда удалить ветку на GitHub проходит чисто:
  git push origin --delete book

  # Состояние:
  # * master
  # remotes/origin/__tempo
  # remotes/origin/master
